---
title: "Get started with Telecast"
description: "Go from zero to a running Telegram channel microblog in minutes — fork the repo, configure your channel, run sync, and build for production."
---

Telecast converts a public Telegram channel into a self-hosted static microblog. This guide walks you through everything from cloning the repository to running a production build.

<Steps>
  <Step title="Check requirements">
    Make sure you have the required tools installed before you begin.

    | Tool | Required version |
    |------|-----------------|
    | Node.js | `>= 22` |
    | pnpm | `>= 10` |

    Verify your versions:

    ```bash
    node --version
    pnpm --version
    ```
  </Step>

  <Step title="Fork and clone the repository">
    Fork [andatoshiki/telecast](https://github.com/andatoshiki/telecast) on GitHub, then clone your fork locally:

    ```bash
    git clone https://github.com/YOUR_USERNAME/telecast.git
    cd telecast
    ```

    All site configuration lives in the repository — there are no external CMS or dashboard. Forking gives you full ownership of your instance.
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure your channel">
    Open `src/lib/constant.ts` and set at minimum the `channel` and `siteUrl` fields:

    ```ts
    export const SITE_CONSTANTS: SiteConstantConfig = {
      channel: 'your_channel',
      locale: 'en',
      timezone: 'UTC',
      siteUrl: 'https://tg.example.com',
      telegramHost: 't.me',
      // ... other options
    }
    ```

    - `channel` — your Telegram channel username without the `@` prefix.
    - `siteUrl` — the canonical URL where the site will be published. Used for RSS, sitemap, and OG tags.
    - `locale` — supported values: `en`, `ja`, `zh`.
    - `timezone` — any IANA timezone string, for example `UTC` or `America/New_York`.

    See [Configuration overview](/configuration/overview) for the full list of options.
  </Step>

  <Step title="Sync channel content">
    Run the sync command to fetch your channel's posts and mirror all media files locally:

    ```bash
    pnpm sync
    ```

    This generates:
    - `src/generated/static-snapshot.json` — page data for all routes
    - `public/search/index.json` — pre-built Lunr full-text search index
    - `public/media/*` — locally mirrored images and videos

    See [Sync command](/features/sync-command) for available flags.
  </Step>

  <Step title="Start the development server">
    ```bash
    pnpm dev:next
    ```

    The development server starts on port `4321`. Open [http://localhost:4321](http://localhost:4321) to preview your site.
  </Step>

  <Step title="Build for production">
    ```bash
    pnpm build
    ```

    <Note>
      `pnpm build` automatically runs `pnpm sync --og-image --favicon` before `next build`. The `--og-image` flag generates an Open Graph image from your channel avatar, and `--favicon` generates `favicon.ico`, `favicon.svg`, and PWA icon PNGs. If you prefer to supply your own graphics, remove the corresponding flags from the `build` script in `package.json` and place your files in `public/` directly.
    </Note>
  </Step>
</Steps>
