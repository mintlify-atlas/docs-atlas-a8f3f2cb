---
title: "SEO and analytics configuration"
description: "Configure meta tags, Open Graph image path, robots noindex/nofollow directives, and Google Analytics or Umami tracking in src/lib/constant.ts."
---

The `seo` and `analytics` objects in `SITE_CONSTANTS` control how your Telecast site appears in search results and social sharing previews, and which analytics providers receive traffic data. Both are optional — leaving all analytics fields empty disables tracking entirely.

## SEO fields

These fields populate the `<head>` meta tags on every page of your site.

<ParamField path="seo.title" type="string" required>
  The site title shown in browser tabs and search result headings. Keep this concise — search engines typically truncate titles beyond 60 characters.

  ```ts
  seo: {
    title: 'My Telecast – Your Name',
  }
  ```
</ParamField>

<ParamField path="seo.description" type="string" required>
  Meta description for search engines and social sharing previews (Twitter cards, Open Graph). Aim for 120–155 characters.

  ```ts
  seo: {
    description: 'Posts and notes from my Telegram channel.',
  }
  ```
</ParamField>

<ParamField path="seo.ogImage" type="string">
  Path to the Open Graph image used for social sharing previews. The recommended value is `/og-auto.png`, which is automatically generated from your channel avatar when you run `pnpm sync --og-image`. To use your own image, place the file in `public/` and set the path here.

  ```ts
  seo: {
    ogImage: '/og-auto.png',
  }
  ```
</ParamField>

<ParamField path="seo.keywords" type="string[]">
  Array of keywords written to the `<meta name="keywords">` tag. Search engines give this limited weight, but it can help with niche indexing.

  ```ts
  seo: {
    keywords: ['telegram', 'microblog', 'your-channel'],
  }
  ```
</ParamField>

<ParamField path="seo.author" type="string">
  Author name written to the `<meta name="author">` tag and used in structured data attribution.

  ```ts
  seo: {
    author: 'Your Name',
  }
  ```
</ParamField>

<ParamField path="seo.noIndex" type="boolean">
  When `true`, emits `noindex` in the robots meta tag. Use this during development or to prevent search engines from indexing your site.

  Defaults to `false`.

  ```ts
  seo: {
    noIndex: false,
  }
  ```
</ParamField>

<ParamField path="seo.noFollow" type="boolean">
  When `true`, emits `nofollow` in the robots meta tag, instructing search engines not to follow links on the page.

  Defaults to `false`.

  ```ts
  seo: {
    noFollow: false,
  }
  ```
</ParamField>

## Analytics fields

Leave any analytics field as an empty string to disable that provider. You can use both providers simultaneously.

<ParamField path="analytics.googleAnalyticsId" type="string">
  Your Google Analytics 4 measurement ID in the format `G-XXXXXXXXXX`. When set, the GA4 script is injected into every page. Leave empty to disable Google Analytics.

  ```ts
  analytics: {
    googleAnalyticsId: 'G-XXXXXXXXXX',
  }
  ```
</ParamField>

<ParamField path="analytics.umamiScriptUrl" type="string">
  The URL of your self-hosted Umami analytics script (e.g. `https://umami.example.com/script.js`). Leave empty to disable Umami.

  ```ts
  analytics: {
    umamiScriptUrl: 'https://umami.example.com/script.js',
  }
  ```
</ParamField>

<ParamField path="analytics.umamiWebsiteId" type="string">
  The website ID from your Umami dashboard for this site. Required when `umamiScriptUrl` is set.

  ```ts
  analytics: {
    umamiWebsiteId: 'xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx',
  }
  ```
</ParamField>

## Example

```ts
export const SITE_CONSTANTS: SiteConstantConfig = {
  // ... identity and display fields

  seo: {
    title: 'My Telecast – Your Name',
    description: 'Posts and notes from my Telegram channel.',
    ogImage: '/og-auto.png',
    keywords: ['telegram', 'microblog', 'your-channel'],
    author: 'Your Name',
    noIndex: false,
    noFollow: false,
  },
  analytics: {
    googleAnalyticsId: '',
    umamiScriptUrl: '',
    umamiWebsiteId: '',
  },

  // ... rest of config
}
```

<Tip>
  The OG image at `seo.ogImage` is auto-generated by `pnpm sync --og-image` using your Telegram channel avatar. If you prefer a custom image, place it in `public/` and update this path, then remove `--og-image` from the `build` script in `package.json`.
</Tip>
